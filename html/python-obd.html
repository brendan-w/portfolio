This python module serves as a back-end for projects using sensor data from a car's OBD-II port. It seemlessly handles the decoding of data from the car, and returns values with units to the programmer. Derived from the popular pyobd project, this package attempts to do a better job of decoupling the serial operations from the UI code.

Since the initial fork of pyobd, the code has been substantially overhauled. The library now makes better use of the Python stdlib, supports more sensors, and presents a simple API for accessing the car's data. Sensors (commands) can now be accessed by name, or Parameter ID. Threaded callbacks can also be subscribed, allowing for asynchronous applications.

Code Examples


import obd

connection = obd.OBD() # auto-connects to USB or RF port

cmd = obd.commands.RPM # select an OBD command (sensor)

response = connection.query(cmd) # send the command, and parse the response

print(response) # "2410 RPM"

This is a simple script that prints the current RPM of the engine. Response objects contain all of decoded information from the request, such as the value and units for the sensor.

Python-OBD also has facilities for asynchronous usage. Particularly useful with UI event loops, Async connections are executed in a seperate thread, and provide event callbacks. The traditional query() function is non-blocking

Sensors

Python-OBD supports a wide variety of sensors. However, because each car has a different payload of those sensors, not all of them may be available. To account for this, the library will automatically request a sensor list from the vehicle, and mark the corresponding commands as "supported". A list of support commands is also stored on the connection object for convenience.

Here are a handful of the sensors that python-OBD supports. For a full list of sensors, see the documentation):

  • Calculated Engine Load
  • Engine Coolant Temperature
  • Fuel Pressure
  • Intake Manifold Pressure
  • Engine RPM
  • Vehicle Speed
  • Timing Advance
  • Intake Air Temp
  • Air Flow Rate (MAF)
  • Throttle Position
  • Engine Run Time
  • Fuel Level Input
  • Barometric Pressure
  • Time since trouble codes cleared
  • Hybrid battery pack remaining life
  • Engine fuel rate

Non-sensor data, such as Diagnostic Trouble Codes (DTCs) can also be retrieved. For instance, connection.query(obd.commands.GET_DTC).value will return a list of every trouble code in the vehicle. Since the trouble codes themselves aren't very meaningful, this library also has a built-in dictionary of code descriptions, which get resolved automatically.
