Collector

Technologies C++, SDL 2.0

Collector is a tag-based file manager for linux. It presents the user's files as a flat data structure, where they can be selected by tag name, or by manual inclusion and exclusion. These tags can be arbitrarily added or removed from files, and can even be queried with logical set operations.

Collector also outputs symlinks of the current selection to a temporary directory, where they can be used by other programs. By default, files are opened in their default programs using the xdg-open utility.

Tags are stored in the file's name and path. This allows directories to function as tags, and helps to represent existing folder structures in Collector. When tags are added or removed, Collector will either splice text from the file name, or move the file to a different place in the directory tree. Not only does this help maintain an organized directory structure, but also prevents the need to synchronize a tag database with the filesystem.
